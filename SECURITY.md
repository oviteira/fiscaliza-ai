# Política de Segurança — TransparênciaIA

## Princípios de Segurança Jurídica

O TransparênciaIA foi desenvolvido com rigoroso cuidado para garantir segurança jurídica em todas as suas operações:

### 1. Dados Exclusivamente Públicos

A plataforma opera **exclusivamente** com dados públicos e abertos, disponibilizados pelos próprios órgãos governamentais. Nenhum dado privado, sigiloso ou protegido por sigilo fiscal, bancário ou funcional é acessado ou processado.

### 2. Linguagem Neutra e Técnica

Todos os alertas e análises utilizam linguagem técnico-estatística neutra:
- **Correto:** "Score de risco elevado", "padrão estatístico atípico", "desvio em relação à mediana"
- **Evitado:** "corrupção", "suspeito", "ilegal", "irregular"

### 3. Rastreabilidade Total

Cada alerta gerado pela plataforma inclui:
- A fonte primária dos dados utilizados
- A metodologia de cálculo aplicada
- O período de referência da análise

### 4. Disclaimer Obrigatório

Todas as páginas com scores ou alertas exibem o seguinte aviso:

> *"Os scores de risco apresentados são calculados por algoritmos estatísticos baseados em padrões de dados públicos e não constituem acusação, imputação de responsabilidade ou afirmação de irregularidade. Os dados são provenientes de fontes governamentais abertas e têm finalidade exclusivamente analítica e de controle social."*

## Reportar Vulnerabilidades

Se você identificar uma vulnerabilidade de segurança nesta plataforma, por favor:

1. **Não abra uma issue pública** com detalhes da vulnerabilidade
2. Envie um e-mail descrevendo o problema
3. Aguarde confirmação antes de divulgar publicamente

## Uso Responsável

Ao utilizar os dados e análises desta plataforma:

- Verifique sempre a fonte primária antes de publicar informações
- Consulte profissionais jurídicos antes de fazer afirmações públicas baseadas nos scores
- Utilize os dados como ponto de partida para investigação, não como conclusão definitiva
- Para investigações formais, acione os órgãos competentes: CGU, TCU, MPF, MPE

## Contato

Para questões de segurança jurídica ou técnica, abra uma issue com o label `security` no repositório.
